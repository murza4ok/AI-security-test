[Мотивация] - этот проект я хочу долго развивать и построить на его основе ядро своего резюме как специалиста ИБ, напоминаю, что я занимаюсь анализом вредоносного программного обеспечения, поиском уязвимостей в Windows, с целью их передачи Microsoft и присвоения CVE,  а также иногда занимаюсь DFIR, когда инцидент сложен для первых линий SOC. Сейчас я хочу попробовать свои силы в новой и развивающейся специализации AI-security.  По проекту уже написана статья, 1 для референса : e:\OBSID_HRAN\ыыыы.md. Сейчас мы на этапе написания второй статьи, для нее есть Roadmap и эта спецификация - свод в единые рамки уже разросшегося проекта.
[Цель] - Создать инструмент, проводящий самостоятельные атаки на LLM-based системы. Предусмотреть генерирующуюся в зависимости от ответа цели полезную нагрузку. Предусмотреть демонстрационный макет, реализующий симуляцию web контура предполагаемой цели, с подключенным LLM-ботом или ассистентом. 
[Состав проекта глобально] - два  проекта. 1-ый - сам инструмент. 2-ой - веб-макет.
[Состав инструмента детально] - инструмент должен включать в себя:
UX - мы уже обсуждали, что терминал должен быть понятным для использования оператором. Должен быть понятный help, описание аргументов и описание режимов работы
generative-mod attacking module - главный модуль, реализован должен быть следующим образом - на базе seed генерирует новые промпты, которые способны вызвать желаемый ответ (а именно выдачу данных БД, выдачу документации, и прочие негативные ответы агента на макете)
seed - исходные промпты и подробное описание атак (это перекочевало из учебного модуля), на основе которых будут генерироваться атакующие запросы к LLM (промпты с полезной нагрузкой)
educational attacking module - изначальный режим атак, который берет только промпты из seed,  "эталонные" промпты. Так же позволяет прочитать описание основных типов атак на LLM.  
[Состав макета детально] - макет должен включать в себя: 
WEB модуль - простое веб приложение, реализующее ИИ-ассистента. Должен принимать на вход запрос от "пользователя" (им является либо атакующий инструмент, либо человек, проверяющий свои промпты). 
Ollama модуль - здесь крутятся разные модели, являющие собой "ядро" LLM-агентов, которые будут отвечать на запросы атакующего инструмента. 
Данные - Бд с пользовательскими данными, БД с сырыми данными, Документация. Это все должно быть симулировано по моему запросу. именно за этими данными и будет "охотиться" инструмент. 
[Ожидания] - от этой итерации я ожидаю демонстрации, что LLM-агенты действительно несут дополнительную поверхность угрозы в себе. Через них, действительно, можно проводить эксфильтрацию данных и возможно запускать вредоносную активность на конечных устройствах пользователей. В этом инструменте, как в предмете обсуждения второй статьи я хочу показать успешную атаку на защищенные макеты, а не просто на модель, у которой есть доступ к сырым данным.
